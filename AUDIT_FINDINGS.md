# Аудит проекта - Найденные проблемы и улучшения

## КРИТИЧЕСКИЕ ПРОБЛЕМЫ

### 1. ❌ Старая система scheduler.py не используется
**Проблема:** В проекте есть `src/scheduler.py` со старой логикой расписания (дни недели: monday, tuesday...), но используется новая система с датами в `src/lessons.py` и `src/recurring_schedule.py`.

**Решение:** Удалить `src/scheduler.py` или полностью переписать под новую систему.

---

### 2. ❌ Напоминания не работают с новой системой
**Проблема:** В `bot_notifications.py` функции напоминаний (`check_and_send_reminders`, `check_and_send_homework_reports`) используют старую систему через `scheduler.py`, которая не совместима с новой системой уроков по датам.

**Решение:** Переписать логику напоминаний:
- Получать уроки из `lessons.py` по дате
- Проверять время урока
- Отправлять напоминания за N минут до урока

---

### 3. ❌ Удаление ученика не удаляет его уроки из календаря
**Проблема:** При удалении ученика через API `/api/students/{user_id}` удаляются только данные ученика, но не его уроки в календаре и не шаблоны расписания.

**Решение:** При удалении ученика также удалять:
- Все его уроки из `lessons.json`
- Все его шаблоны из `recurring_schedule.json`

---

### 4. ❌ Удаление шаблона не удаляет будущие уроки
**Проблема:** Когда удаляешь шаблон расписания, уже созданные уроки в календаре остаются. Это может быть нелогично - если репетитор удалил постоянный урок, он скорее всего хочет удалить и все будущие уроки.

**Решение:** При удалении шаблона предлагать:
- Удалить только шаблон (будущие уроки останутся)
- Удалить шаблон + все будущие незавершенные уроки

---

## ВАЖНЫЕ УЛУЧШЕНИЯ

### 5. ⚠️ Нет проверки конфликтов при добавлении урока
**Проблема:** Можно добавить два урока на одно и то же время. В коде есть проверка `if lesson_id in lessons` но это проверяет только точное совпадение (дата + время + ученик). Можно добавить разных учеников на одно время.

**Решение:** При добавлении урока проверять что на это время нет других уроков (любого ученика).

---

### 6. ⚠️ Нет возможности отменить урок
**Проблема:** Есть только "Удалить урок" и "Перенести урок". Но в реальности урок может быть отменен (ученик заболел, репетитор занят). Отмененный урок != удаленный урок (для статистики важно знать сколько было отмен).

**Решение:** Добавить статус урока:
- `scheduled` - запланирован
- `completed` - проведен
- `cancelled` - отменен
- `moved` - перенесен (уже есть через `is_moved`)

---

### 7. ⚠️ Нет истории изменений урока
**Проблема:** Если урок перенесли несколько раз, нет истории. Не видно когда и куда переносили.

**Решение:** Добавить массив `history` в урок с записями о всех изменениях.

---

### 8. ⚠️ Статистика не учитывает отмененные уроки
**Проблема:** В статистике показывается "Проведено" и "Запланировано", но нет "Отменено". Для репетитора важно знать процент отмен.

**Решение:** Добавить в статистику:
- Отмененные уроки
- Процент отмен
- Причины отмен (если указаны)

---

### 9. ⚠️ Нет фильтрации уроков по ученику
**Проблема:** В календаре показываются все уроки всех учеников. Если учеников много, сложно найти уроки конкретного ученика.

**Решение:** Добавить фильтр по ученику в календаре.

---

### 10. ⚠️ Нет возможности изменить время/дату урока напрямую
**Проблема:** Чтобы изменить время урока, нужно его перенести (выбрать дату, выбрать слот). Если нужно просто сдвинуть на 30 минут, это неудобно.

**Решение:** В модальном окне урока добавить кнопку "Изменить время" с простым выбором новой даты и времени.

---

## СРЕДНИЕ УЛУЧШЕНИЯ

### 11. 📝 Нет заметок к уроку
**Проблема:** Репетитор не может добавить заметку к уроку (что проходили, что задали, особенности).

**Решение:** Добавить поле `notes` к уроку.

---

### 12. 📝 Нет возможности добавить несколько уроков сразу
**Проблема:** Если нужно добавить 5 уроков подряд с одним учеником, приходится 5 раз открывать модальное окно.

**Решение:** Добавить опцию "Добавить еще урок" после создания урока.

---

### 13. 📝 Нет поиска по ученикам
**Проблема:** Если учеников много (50+), сложно найти нужного в списке.

**Решение:** Добавить поле поиска на странице "Ученики".

---

### 14. 📝 Нет экспорта данных
**Проблема:** Репетитор не может экспортировать данные (для бухгалтерии, отчетов).

**Решение:** Добавить кнопку "Экспорт" с форматами:
- Excel (уроки за месяц)
- PDF (отчет за месяц)
- CSV (все данные)

---

### 15. 📝 Нет напоминания репетитору о незавершенных уроках
**Проблема:** Если репетитор забыл отметить урок как выполненный, нет напоминания.

**Решение:** Вечером отправлять список незавершенных уроков за день.

---

### 16. 📝 Нет возможности массовых операций
**Проблема:** Нельзя выбрать несколько уроков и выполнить действие (отметить все как выполненные, удалить все).

**Решение:** Добавить чекбоксы для выбора уроков и кнопки массовых действий.

---

### 17. 📝 Нет уведомления ученику об отмене урока
**Проблема:** Если репетитор отменяет урок, ученик не получает уведомление.

**Решение:** При отмене/переносе урока отправлять уведомление ученику в Telegram.

---

### 18. 📝 Нет возможности ученику отменить урок
**Проблема:** Ученик не может отменить урок через бота (только репетитор через веб).

**Решение:** Добавить в бот кнопку "Отменить урок" для ученика.

---

### 19. 📝 Нет резервного копирования
**Проблема:** Все данные в JSON файлах. Если файл повредится, все потеряется.

**Решение:** Добавить автоматическое резервное копирование:
- Ежедневно сохранять копию всех JSON файлов
- Хранить последние 7 дней

---

### 20. 📝 Нет валидации времени урока
**Проблема:** Можно создать урок на 25:00 или на прошедшую дату.

**Решение:** Добавить валидацию:
- Время в формате HH:MM (00:00 - 23:59)
- Дата не в прошлом (кроме ручного добавления)

---

## МЕЛКИЕ УЛУЧШЕНИЯ

### 21. 💡 Нет индикатора загрузки
**Проблема:** При загрузке данных нет индикатора, непонятно загружается или зависло.

**Решение:** Добавить спиннер загрузки.

---

### 22. 💡 Нет подтверждения при удалении
**Проблема:** Есть `confirm()` но можно случайно нажать OK.

**Решение:** Добавить более заметное модальное окно подтверждения.

---

### 23. 💡 Нет горячих клавиш
**Проблема:** Все действия только мышью.

**Решение:** Добавить горячие клавиши:
- `N` - новый урок
- `T` - сегодня
- `Esc` - закрыть модальное окно

---

### 24. 💡 Нет темной темы
**Проблема:** Только светлая тема.

**Решение:** Добавить переключатель темы.

---

### 25. 💡 Нет мобильной версии бота для репетитора
**Проблема:** Репетитор может управлять только через веб. Если нет компьютера, неудобно.

**Решение:** Добавить команды в бот для репетитора:
- Посмотреть расписание на день
- Отметить урок как выполненный
- Отменить урок

---
