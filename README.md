# Telegram Бот для Репетитора

Система управления расписанием уроков с веб-панелью и Telegram ботом.

## Возможности

### Веб-панель для репетитора:
- Дашборд со статистикой (уроки, доход)
- Управление расписанием на неделю
- Управление учениками
- Настройка цен за урок
- Уведомления в реальном времени (WebSocket)
- Настройки системы

### Telegram бот для учеников:
- Регистрация с выбором часового пояса
- Просмотр своего расписания
- Автоматические напоминания об уроках
- Отчет о выполнении домашнего задания

### Telegram бот для репетитора:
- Просмотр расписания на сегодня
- Ссылка на веб-панель

## Установка локально

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Создайте файл `.env`:
```
BOT_TOKEN=ваш_токен_от_BotFather
ADMIN_ID=ваш_telegram_id
PORT=8000
WEB_URL=http://localhost:8000
WEB_SERVER_URL=http://localhost:8000
```

3. Запустите систему:
```bash
python start.py
```

4. Откройте веб-панель: http://localhost:8000

## Деплой на Railway

### Шаг 1: Подготовка проекта

1. Убедитесь что все файлы закоммичены в git:
```bash
git add .
git commit -m "Prepare for Railway deployment"
git push
```

### Шаг 2: Создание проекта на Railway

1. Зайдите на [railway.app](https://railway.app)
2. Нажмите "New Project"
3. Выберите "Deploy from GitHub repo"
4. Выберите ваш репозиторий

### Шаг 3: Настройка переменных окружения

В настройках проекта Railway добавьте переменные:

```
BOT_TOKEN=ваш_токен_от_BotFather
ADMIN_ID=ваш_telegram_id
WEB_URL=https://ваш-проект.railway.app
WEB_SERVER_URL=http://localhost:8000
```

**Важно:** `WEB_URL` нужно будет обновить после первого деплоя, когда Railway выдаст вам домен.

### Шаг 4: Деплой

1. Railway автоматически задеплоит проект
2. После деплоя скопируйте URL вашего приложения (например: `https://your-app.railway.app`)
3. Обновите переменную `WEB_URL` на этот URL
4. Railway автоматически передеплоит проект

### Шаг 5: Проверка

1. Откройте ваш URL в браузере - должна открыться веб-панель
2. Напишите боту в Telegram `/start` - бот должен ответить

### Особенности Railway

- Railway автоматически определяет порт через переменную `PORT`
- Проект использует `asyncio` вместо `multiprocessing` для совместимости с контейнерами
- WebSocket работает из коробки
- Данные хранятся в папке `data/` (будут сброшены при рестарте, для продакшена используйте БД)

### Troubleshooting

**Бот не отвечает:**
- Проверьте что `BOT_TOKEN` правильный
- Проверьте логи в Railway Dashboard

**Веб-панель не открывается:**
- Проверьте что проект задеплоился успешно
- Проверьте логи на ошибки

**Уведомления не приходят:**
- Проверьте что `WEB_SERVER_URL=http://localhost:8000` (внутри контейнера)
- Проверьте что WebSocket подключается (в консоли браузера)

## Получение токена

1. Найдите [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте `/newbot`
3. Следуйте инструкциям
4. Скопируйте токен

## Получение Telegram ID

1. Найдите [@userinfobot](https://t.me/userinfobot)
2. Отправьте `/start`
3. Скопируйте ваш ID

## Настройки

Настройки доступны в веб-панели в разделе "Настройки":
- Часовой пояс репетитора (по умолчанию МСК)
- За сколько минут напоминать об уроке
- За сколько минут проверять ДЗ
- Время ежедневного напоминания
- Цена урока по умолчанию

## Часовые пояса

Система автоматически учитывает часовые пояса:
- Репетитор ведет расписание в своем времени
- Каждый ученик указывает свой часовой пояс при регистрации
- Все уведомления приходят в правильное время для каждого
- Время отображается относительно МСК (Московское время)
